## What's new

### 1. Siswa: rekap nilai per-kelas + lintas kelas (Task 7.A)

Endpoints:
- `GET /api/v1/siswa/kelas/:id/nilai` — per-kelas nilai breakdown (bab + ulangan harian + ujian).
- `GET /api/v1/siswa/nilai` — cross-class summary.

Pages:
- `/siswa/kelas/[id]/nilai` — transparant per-bab breakdown dgn formula breakdown box.
- `/siswa/nilai` — list semua kelas siswa + total per-kelas.

Locked decision #90: read-only at-query-time aggregator (no
`nilai_*` denormalized tables). Performance bound: per-kelas typical
< 50 siswa × < 20 bab — single-pass JOIN sufficient.

### 2. Guru: rekap matrix + CSV export (Task 7.B)

Endpoint:
- `GET /api/v1/kelas/:id/rekap?format=json|csv` — siswa × bab matrix
  + ulangan harian + ujian + total per-kelas.

CSV format: UTF-8 BOM (Excel-compat), columns
`nis,nama,<bab1_total>,...,<ujian1_total>,...,total_kelas`.
Filename `rekap-<kelas-id>-<YYYYMMDD>.csv`. Sync inline,
< 500ms expected for ≤200 siswa × ≤30 kolom.

Page: `/guru/kelas/[id]/rekap` — color-tier (≥75 emerald / ≥60
amber / <60 rose) + Download CSV button.

### 3. Activity feed guru (Task 7.C)

Endpoint:
- `GET /api/v1/guru/feed?cursor=&limit=` — opaque base64 cursor
  pagination `(at_unix_micro DESC, id DESC)` per locked #55.

Source: UNION ALL 3 events — `submission_baru`, `ulangan_selesai`,
`siswa_join`. (Locked #92 mentioned 6 events — MVP shipped 3, future
v0.14+ enhancement.)

Component: `<GuruFeedList>` di dashboard `/guru` "Aktivitas terbaru"
card — `useInfiniteQuery` + 30s polling + per-event icon/link +
late/nilai badges + load-more.

Limit clamp [1, 50]. Default 20. **v0.13.0 fix**: invalid `limit`
(parse fail or n<1) now returns `400 invalid_limit` (was silent
fallback to default).

### 4. Pending counters consolidated (Task 7.D)

Endpoint:
- `GET /api/v1/guru/pending-counts` — single endpoint, 3 fields:
  `ungraded_submissions`, `pending_review_ulangan`,
  `pending_review_ujian`.

Implementation: 3 SQL aggregates parallel via `sync.WaitGroup`
sharing kelas-IDs filter. Admin = no kelas filter (sees all).

UI:
- Sidebar badge: `pending_total` (sum 3 counters), 30s polling.
- Dashboard `/guru` 4-card grid (Total Kelas, Tugas perlu dinilai,
  Review ulangan, Review ujian).

### 5. Guru audit scope per-kelas (Task 7.E)

Endpoints:
- `GET /api/v1/guru/kelas/:id/audit?action=&limit=&offset=` —
  per-kelas audit log dgn ownership guard + 47-action allowlist
  defense-in-depth.
- `GET /api/v1/guru/audit-actions` — allowlist actions for FE
  filter dropdown.

Hard scope `WHERE target_kelas_id=:id`. Cross-kelas → 403. Admin
bypass via role check. Pagination: offset-based 50/page.

Page: `/guru/kelas/[id]/audit` — native `<select>` filter + 44
ACTION_LABEL Bahasa entries + offset pagination.

### 6. UX/QA pass (Task 7.F)

Static FE↔BE contract drift audit (browser unavailable Windows).
5 findings: 0 Critical, 2 High (feed limit silent fallback), 2
Medium (audit error envelope drift + error key naming convention),
1 Low (refetchIntervalInBackground).

**Fixed in this release**:
- HIGH: `feed/handler.go` strict limit parsing (parse fail / n<1
  → 400).
- MEDIUM: `audit/handler.go` error envelope aligned to project
  default `{error, code, request_id}` — was `{error, message}`,
  missing `request_id` (debugging harder).
- LOW: `guru/page.tsx` pendingQ defensive `refetchIntervalInBackground:
  false`.

**Deferred to v0.14.0**: error key naming convention drift
(`invalid_id` vs `invalid_kelas_id` across nilai/feed/submission/audit
handlers) — multi-file blast radius, no behavioral break.

## Technical notes

- **Postgres quirk locked**: `MAX(uuid)` SQLSTATE 42883 not supported
  → 2-CTE pattern (agg `MAX/COUNT` non-uuid + `DISTINCT ON ujian_id
  ORDER BY at DESC`).
- **Schema findings**: `tugas` no `deleted_at` & no `max_nilai`
  (uses `nilai_setelah_penalty` NUMERIC(5,2) + status='published'
  filter). `hasil_ujian` has `deleted_at` + status enum.
- **Static export FE**: dynamic routes use `?id=:id` query-string
  pattern (Next.js static export limitation).
